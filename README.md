# CS Portfolio – Artemis Financial Project Reflection

## Briefly summarize your client, Artemis Financial, and its software requirements.
Artemis Financial was the client for this project. They are a financial company that needed stronger software security to protect sensitive customer and business information. The main issue they wanted addressed was identifying vulnerabilities in their software and improving security practices to reduce risks.

## What did you do well when you found your client’s software security vulnerabilities? Why is it important to code securely?
I think I did well reviewing the system for possible risks and identifying areas that needed stronger protection. It is important to code securely because it helps prevent data breaches, protects customer information, and keeps systems running safely. Good software security helps build trust with customers and protects the company’s reputation.

## Which part of the vulnerability assessment was challenging or helpful to you?
One challenging part was understanding all the different vulnerabilities and how they could affect the system. A helpful part was learning how security tools can scan software and identify weaknesses quickly.

## How did you increase layers of security? In the future, what would you use to assess vulnerabilities?
I increased layers of security by recommending secure coding practices, encryption, dependency checks, and regular updates. In the future, I would continue using vulnerability scanning tools, code reviews, and risk assessments to decide the best mitigation techniques.

## How did you make certain the code and software application were functional and secure?
I made sure the software was functional and secure by testing the program after changes were made and reviewing the results for errors. After refactoring the code, I checked for new vulnerabilities by rerunning security scans and making sure the application still worked correctly.

## What resources, tools, or coding practices did you use that might be helpful in future assignments or tasks?
Some helpful resources and tools were dependency-check tools, secure coding practices, Java development tools, and vulnerability reports. These tools can help identify risks early and improve software quality.

## What might you show future employers from this assignment?
I could show future employers this project as an example of my ability to analyze software security risks, recommend solutions, and apply secure coding practices. It also shows problem-solving and attention to detail.
